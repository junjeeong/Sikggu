# SIKGGU Product Requirements Document (PRD)

## 1. Executive Summary

**SIKGGU(식구)**는 유통기한 임박/B급 신선식품을 위치 기반 실시간 타임세일로 연결하는 **동네 기반 하이퍼로컬 O2O 커머스 플랫폼**.
- 동네 마트의 악성 재고(폐기 비용) 문제 해결
- 1인 가구의 식비 부담 문제 해결
- **선결제 → 후픽업** 구조로 노쇼 리스크 제거

---

## 2. 타겟 고객 및 Pain Point

| 고객군 | Persona | Pain Point |
|--------|---------|------------|
| 공급자 (B) | 동네 마트, 청과물점, 반찬가게 사장님 | 신선식품 반품 불가 → 폐기 손실. 온라인 커머스 입점은 높은 수수료와 복잡한 IT |
| 수요자 (C) | 2030 1인 가구, 자취생 | 물가 상승 → 식자재 구매 부담. 대용량 위주 판매 → 소량 구매 어려움 |

### 가치 제안
- **사장님**: 버려질 재고를 '단 1분의 조작'으로 현금화 (Zero 폐기율)
- **자취생**: 퇴근길 동네 마트에서 신선 식재료를 최대 70% 할인에 즉시 픽업

---

## 3. MVP 핵심 KPI (출시 후 4주 내)

| 지표 | 목표 |
|------|------|
| 공급 (Supply) | 타겟 테스트베드 내 제휴 가맹점 **5곳** |
| 수요 (Demand) | 초기 가입 유저 **100명**, WAU **100명** |
| 핵심 전환 | 타임세일 상품의 당일 결제 완료율(재고 소진율) **50% 이상** |

### Go-To-Market (초기 100명 모객)
- **테스트베드**: 자취생 비율 높은 특정 동네 (예: 서울시 관악구 신림동)
- **사장님 영업**: PM이 직접 방문, "앱 수수료 0원, 폐기 비용 절감" 소구
- **유저 모객**: 당근마켓 '동네생활' 바이럴 + 제휴 마트 입구 QR코드 전단지

## 4. 화면 정의 (Information Architecture)

### 공통
- Splash & Onboarding: 서비스 소개 3컷 + 로그인/회원가입

### 구매자 (유저) App
| 화면 | 설명 |
|------|------|
| Home (지도/리스트) | 내 주변 반경 2km 마감 할인 상품 목록 (정렬/필터) |
| Product Detail | 상품 상세, B급 사유, 잔여 수량, 남은 시간, [찜하기/결제하기] |
| Checkout | 간편 결제 창 (포트원/토스 연동) |
| My Page / Receipt | 결제 내역, 픽업 상태, 예약 번호(4자리) 확인 |

### 판매자 (사장님) App
| 화면 | 설명 |
|------|------|
| Dashboard (Home) | 진행 중인 타임세일 목록 + 픽업 대기 건수 요약 |
| Product Register | 타임세일 상품 간편 등록 폼 (사진, 이름, 원가/할인가, 수량) |
| Pickup Management | 예약 번호(전화번호 뒷자리) 검색 + [수령 완료] 처리 |

---

## 5. User Stories

- **구매자**: 퇴근길에 내 주변 2km 이내 마감 할인 상품을 확인하고, 앱 내에서 즉시 결제하여 재고를 선점할 수 있다. 찜하기를 누르면 마감 30분 전에 푸시 알림을 받을 수 있다.
- **판매자**: 복잡한 절차 없이 상품명, 가격, 수량만 입력해 즉시 세일을 시작할 수 있고, 손님이 오면 구두로 전달받은 번호만 누르고 '수령 완료' 처리를 할 수 있다.

## 7. 비기능 요구사항 (NFR)

| 항목 | 요구사항 |
|------|---------|
| 응답 속도 | 홈 화면 상품 목록 조회(LBS 기반) **2초 이내** |
| 동시성 처리 | 수량 1개에 100명 동시 결제 시 초과 결제(Overbooking) **0%** — DB 비관적 락 필수 |
| 가용성 | 결제/픽업 화면은 오프라인 마트 내부에서도 안정적 작동 또는 재시도 로직 포함 |

## 8. 예외 처리 및 운영 정책

### 픽업 인증
- QR코드 방식 배제 (바쁜 마트 계산대 현실 반영)
- 유저가 **예약 번호 4자리** 또는 **전화번호 뒷자리**를 구두 전달 → 사장님이 번호 대조 후 터치 한 번으로 인증

### 환불 정책
| 시점 | 정책 |
|------|------|
| 픽업 마감 1시간 전까지 | 100% 환불 가능 |
| 마감 1시간 이내 | 환불 불가 |
| 픽업 시간 경과 후 미방문 (노쇼) | 전액 환불 불가 (사장님 리스크 Zero) |

### 동시 예약 충돌 방지
- DB 비관적 락 또는 트랜잭션 처리
- 결제창 진입 시 임시 홀딩, 제한 시간 내 미결제 시 롤백

### 시간 만료 스케줄링
- 마감 시간 경과 → 자동으로 앱 노출 차단 + 상태 `EXPIRED` 변경
